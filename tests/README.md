# בדיקות-רגרסיה למנוע ההכרעה

קורפוס מתויג שמוודא שמנוע ההכרעה (`fallbackVerdict` ב-`components.jsx`) מסווג
נכון מקרים אמיתיים — גם תקלות וגם לא-תקלות. כל שינוי במנוע חייב לשמור את כל
המקרים ירוקים.

## הרצה
```bash
node tests/run.mjs
```
(מחזיר exit-code 1 אם מקרה כלשהו נכשל — מתאים ל-CI.)

## הוספת מקרה
1. ייצאו את הקו הבעייתי מהאפליקציה (כפתור הייצוא של הקו → `kavbug-קו-XXX...json`).
2. שמרו אותו ב-`tests/cases/` בשם תיאורי (למשל `haifa-line-12.json`).
3. הוסיפו שורה ל-`tests/labels.json` עם הסיווג הנכון (`expect`) והערה קצרה.
4. הריצו `node tests/run.mjs` ואמתו שהוא ירוק.

## הסיווגים האפשריים
- `אמיתי` — עיקוף ממשי (באג מסלול).
- `כיסוי לגיטימי` — סטייה מוצדקת / מבנה-כביש.
- `רעש` — תנודה זעירה / מסלול ישר.
- `לא ניתן להשוואה` — קו-הייחוס אינו תקף (כיוון מנוגד, ציר זר, תחנת-קצה…).
- `ספק` — דורש בדיקה אנושית.

> ה-harness טוען את פונקציות-ההכרעה ישירות מ-`components.jsx` (מקור-אמת יחיד),
> כך שהוא בודק את הקוד החי ולא עותק.
